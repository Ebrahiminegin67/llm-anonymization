Verdict — Does Either Architecture Improve Anonymization?

1 · Adversarial Accuracy

All architectures reduce adversarial accuracy by the same 10 percentage points (65% → 55%). The best single view is Baseline at 55.0%, which equals the baseline. On the paper's primary metric, neither parallel nor sequential produces a statistically meaningful improvement.

2 · Evidence Rate — the clearest difference

The best post-anonymization evidence rate belongs to Parallel B at 30.0%, compared with the baseline's 50.0%. Both parallel and sequential anonymizers suppress implicit signals (style-based and informed cues) that the single-model baseline never knew to target. Sequential Attack B benefits the most: it was explicitly told what Attack A found, so when the anonymizer erases those signals, B can no longer find them.

3 · Utility — no trade-off penalty

All three architectures deliver essentially the same combined utility (baseline 0.942, parallel 0.941, sequential 0.937). Neither advanced architecture sacrifices text quality to achieve better privacy.

Summary

  • Adversarial accuracy (paper's primary metric): no improvement — all architectures tie at 55% post-anon Top-3.
  • Evidence rate (secondary metric): parallel B −20pp, sequential B −15pp vs baseline's drop — real but doesn't translate to lower accuracy.
  • Utility: all three are equivalent (~0.94).
  • Caveat: n=20 profiles / 20 PII instances. A 10pp accuracy difference requires ~100 instances for 80% power. Results are directionally informative but not conclusive.
