Applying the three core metrics from "LLMs Are Advanced Anonymizers" to the parallel inference pipeline (GPT-4o analytical + Claude sociolinguistic). Evaluated on 20 PII-type instances from 14 profiles.

Accuracy method: String-similarity matching (Jaro–Winkler + normalised label matching for ordinal types) ; the same approach used in the paper's automated evaluation. No LLM judge is called here, which gives a conservative lower bound on true accuracy; the paper additionally uses GPT-4 as a judge for edge cases, which would push accuracy slightly higher.

Key Findings — How the Parallel Attack Affected Anonymization

1. Paper's Primary Metric (Top-3 Accuracy) — No Improvement

Top-3 adversarial accuracy: 65% → 55% — identical to the single-model baseline. Using two attackers simultaneously did not reduce how often the attacker finds the correct answer within 3 guesses. Both baseline and parallel converge at the same 55% floor.

2. Top-1 Accuracy — Small Improvement over Baseline

GPT-4o Top-1: 50% → 35% (▼ 15pp)  |  Claude Top-1: 45% → 30% (▼ 15pp)  |  Baseline Top-1: 50% → 40% (▼ 10pp only). The parallel anonymizer, having seen two reasoning chains, reduced first-guess accuracy more aggressively than baseline (−15pp vs −10pp). The attacker's single best guess became less reliable — but it could still find the answer within 3 guesses just as often.

3. Evidence Rate — The Clearest and Strongest Improvement

Claude evidence rate: 55% → 30% (▼ 25pp)  |  GPT-4o evidence rate: 60% → 50% (▼ 10pp)  |  Baseline: 60% → 50% (▼ 10pp). This is the most significant result. The parallel anonymizer specifically removed the sociolinguistic signals Claude relies on — cutting its evidence rate by 25pp compared to baseline's 10pp. After anonymization, Claude is mostly guessing rather than reading direct text evidence. However, this did not translate into lower Top-3 accuracy — the attacker still guesses correctly through demographic inference.

4. Utility — No Tradeoff

Combined Utility: 0.941 (Readability 10.0/10  ·  Meaning 8.9/10  ·  ROUGE-1 0.931). Baseline utility: 0.942 — difference of 0.001, negligible. The parallel pipeline's additional complexity — showing the anonymizer two reasoning chains instead of one — did not damage the text. No privacy–utility tradeoff was necessary.

Summary

The parallel attack made the anonymizer smarter about what to edit — it removed more implicit sociolinguistic signals (evidence rate −25pp for Claude) and reduced first-guess accuracy more aggressively (−15pp vs −10pp baseline). However, it could not address signals encoded in how a person writes rather than what they explicitly say — vocabulary register, discourse patterns, topic selection — which keeps Top-3 accuracy at 55% regardless of architecture. Utility was unaffected at 0.941.
