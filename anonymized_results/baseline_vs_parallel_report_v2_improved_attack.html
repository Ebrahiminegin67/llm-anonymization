Key Findings

1. The parallel pipeline defeats attacks 3x more often: 45.0% vs 15.0%

The merged inference (combining GPT-4o analytical + Claude sociolinguistic) gives the anonymizer significantly better information about what to protect. When the anonymizer knows about more attack vectors, it removes more identifying information.

2. The biggest v2 improvement: anonymizer aggressiveness

In v1, the average certainty drop was only 0.15 for both attacks — the anonymizer barely reacted. In v2, the drop is 0.50 (A) and 0.70 (B), a 3–5x increase. The key change: the anonymizer now receives both inference perspectives and an explicit warning when two independent attacks agree, prompting more thorough rewrites.

3. Attack B is now genuinely contributing

In v1, Attack B (Claude) averaged certainty 1.5 and failed on 7/20 profiles. In v2, it averages 2.45 — nearly matching Attack A's 2.60 — with only 6 MISSING responses. The stricter prompt (no markdown, limited preamble, unambiguous certainty format) eliminated most parsing failures.

4. Defeat rate is the more meaningful metric

Certainty drop is continuous and based on model self-assessment. “Defeated” means the attacker's guesses completely changed — the anonymizer successfully misdirected the attack. 45.0% defeated vs 15.0% defeated is a clear win for the parallel approach.

Bottom Line

The parallel pipeline (v2) produces meaningfully better anonymization than both the baseline and v1. The defeat rate is 3x higher than baseline (45% vs 15%), and the certainty drop improved 3–5x over v1 (0.15 → 0.50/0.70). The two key drivers: a stronger Attack B prompt that actually parses correctly, and an anonymizer that now knows it must defeat two independent attack perspectives simultaneously.

Methodology Notes

  • Baseline: Standard sequential pipeline — single GPT-4o inference → anonymize → utility → re-inference (2 iterations)
  • Parallel v2: GPT-4o (analytical) + Claude Sonnet (sociolinguistic, strict format) parallel inference → certainty-ordered merge → anonymize with dual-inference context → utility → parallel re-inference
  • Defeated: No overlap between pre- and post-anonymization guesses (attacker completely misdirected)
  • Agreement: Whether the two parallel attacks agree on guesses before anonymization
  • Same data: Both pipelines ran on the same 20 profiles from inference_0.jsonl with identical filters
