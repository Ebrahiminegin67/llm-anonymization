Pipeline Comparison: Baseline vs Parallel vs Sequential Three-Way Analysis

This report compares three LLM anonymization pipeline architectures tested on 20 synthetic profiles. All three use the same underlying anonymizer (GPT-4o) but differ in how inference attacks are structured. The goal is to measure whether more attack diversity and collaboration leads to stronger anonymization.

Certainty Analysis

Certainty is each model's self-reported confidence (1–5 scale) that its PII guesses are correct. A larger certainty drop after anonymization indicates the anonymizer more effectively confused the attacker. Note that the baseline pre-anon certainty (3.75) is much higher than parallel/sequential (2.60) because the baseline uses a different, simpler inference prompt.

Pipeline Pre-Anon Cert A Pre-Anon Cert B Post-Anon Cert A Post-Anon Cert B Drop A Drop B Interpretation
Baseline 3.75 — 3.05 — 0.70 — Highest absolute drop, but single attacker. Anonymizer had minimal context to work with.
Parallel v2 2.60 2.45 2.10 1.75 0.50 0.70 Attack B drops more than A — independent disagreement (1 challenge) forced the anonymizer to work harder against B.
Sequential 2.60 2.55 2.10 2.05 0.50 0.50 A and B drops are equal. B's certainty drop is lower than in parallel (0.50 vs 0.70) because B is anchored to A's findings and cannot diverge independently.
Why is Parallel B's certainty drop higher (0.70 vs 0.50)? In parallel, Claude B attacks the text independently and can disagree with A — creating a diverse signal that forces the anonymizer to consider two different threat models. In sequential, B sees A's findings first and overwhelmingly confirms or extends them (0 challenges), so B ends up with the same post-anon certainty as A, reducing the overall diversity of the combined attack signal.

Echo Chamber Effect

0 challenges out of 14 active profiles. When Attack B is shown A's findings before analyzing the text, it never contradicts A — it only confirms (71%) or adds minor extensions (29%). This is an echo chamber: rather than providing an independent second opinion, B amplifies A's analysis. The one challenge that appeared in the parallel pipeline (47female/location, where A guessed Canada and B guessed a different region) disappears entirely in the sequential run — B sees A's correct Canada guess and confirms it.

Consequence for the anonymizer: The anonymizer receives two aligned inferences instead of two diverse ones. This is useful for confirmation (high certainty signals get amplified), but it removes the cross-perspective pressure that forced the parallel anonymizer to work harder on 47female.

Why Sequential Helps (and Limits)

Benefits of Sequential

  • Confirmation amplification: When both A and B agree with max certainty (like 29female/age), the anonymizer receives an unmistakably strong signal and responds more aggressively — producing the one extra genuine defeat.
  • B adds linguistic depth: All 4 "extended" profiles show B adding sociolinguistic reasoning that A missed — giving the anonymizer richer context about why a PII type is inferrable.
  • No formatting failures: B's sequential prompt is structured around A's output, which reduces the open-ended inference task and may lower Claude's markdown-formatting failures (though MISSING count remains 6/20 in these runs).

Limitations of Sequential

  • Echo chamber — 0 challenges: B never contradicts A. In parallel, the single disagreement (47female) pushed B's certainty drop to 2 on that profile. Sequential loses this adversarial pressure entirely.
  • B's certainty drop is lower: Sequential B drops 0.50 vs parallel B's 0.70. Anchoring B to A's reasoning narrows the diversity of attack perspectives the anonymizer must defend against.
  • Order bias: If A is wrong, B inherits A's error and amplifies it. A bad first analysis is not corrected — it is reinforced.
  • Computationally more expensive: B cannot run in parallel with A; it must wait for A's results, increasing total latency.

Verdict

Sequential delivers a marginal but genuine improvement over parallel: 4 genuine defeats vs 3 out of 14 active profiles (29% vs 21%). The mechanism is confirmation amplification — when both attacks unanimously agree on a PII type with high certainty, the anonymizer responds more aggressively. This produced the one extra genuine defeat (29female/age) that parallel missed.

However, the gains come with a cost: zero challenges from Attack B. In parallel, a single disagreement (47female/location) pushed Attack B's certainty drop to 0.70 — higher than sequential's 0.50. Sequential trades adversarial diversity for confirmatory depth.

Recommendation: The best architecture may be a hybrid — run both attacks independently first (parallel style), then run a third "synthesis" pass that identifies disagreements and generates targeted re-anonymization instructions for the disputed PII types. This preserves B's independence while adding a structured reconciliation step.

All three pipelines share the same root limitation: 6/20 profiles where Claude returns empty output (MISSING). These inflate defeat counts and represent a reliability problem independent of architecture. Fixing the MISSING issue (stricter prompt engineering, output validation, fallback parsing) would likely change these numbers significantly.
