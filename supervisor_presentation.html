Master's Thesis Research

Evaluating Multi-Model Adversarial Attack
Architectures for LLM-Based Anonymization

An empirical study extending the Adversarial Anonymization (AA) framework from Staab et al. (ICLR 2025) by replacing the single-model attacker with parallel, sequential, evidence-targeted, and multi-round multi-model attack configurations.

Dataset: 20 Synthetic Profiles
Attacker A: GPT-4o (Analytical)
Attacker B: Claude Sonnet 4.6 (Sociolinguistic)
Anonymizer: GPT-4o
Pipelines Tested: 5

The Original Paper

What we are extending — Staab et al., "Language Models are Advanced Anonymizers", ICLR 2025

Core Idea: Adversarial Anonymization (AA) Framework

Instead of anonymizing text blindly, use an LLM as an attacker — ask it "what can you infer about this person?" — then show its reasoning to an anonymizer and say "now fix these leaks." Re-attack to verify. Repeat.

📊 Paper's Key Result

Single GPT-4 attacker over 5 rounds reduces adversarial accuracy from 86.7% → 41.6%. Filtering to high-confidence predictions (certainty ≥ 3) drops it further to 7.7%.

🔮 Paper's Open Question

The paper explicitly states in its future work (Section G.2): "An interesting direction would be to use multiple models as attackers, since different models might identify different types of evidence."

🎯 Our Thesis Question

Does using two LLMs with different reasoning styles (GPT-4o analytical + Claude sociolinguistic) as the attacker produce better anonymization than a single model?

Dataset & Experimental Setup

What we ran experiments on

Data

  • 20 synthetic Reddit-style user profiles
  • Source: paper's own benchmark dataset
  • Filter: hardness ≥ 1, certainty ≥ 1, max 2000 tokens
  • 525 total profiles available; 20 used as pilot study

PII Attributes Targeted

  • Age  ·  Gender  ·  Income
  • Education  ·  Marital Status
  • Occupation  ·  Location
  • Total: 20 inferences evaluated per run

Models Used

  • Attacker A: GPT-4o (temp 0.1) — analytical, structured
  • Attacker B: Claude Sonnet 4.6 (temp 0.5) — sociolinguistic
  • Anonymizer: GPT-4o (temp 0.1)
  • Utility Judge: GPT-4o (temp 0.1)

Design Principle

  • All new code in separate files — original paper code untouched
  • Same metrics as paper for fair comparison
  • Each pipeline run independently with its own output directory

Evaluation Metrics

Three metrics from the paper, one derived from the paper's framework

🎯

Adversarial Accuracy Top-3

Fraction of PII attributes where the correct answer appears in the attacker's top-3 guesses. Primary paper metric.

↓ Lower is better
🎯

Adversarial Accuracy Top-1

Fraction where the attacker's single best guess is correct. Stricter than Top-3.

↓ Lower is better
🔍

Evidence Rate

Fraction of predictions where the attacker cites direct text evidence (certainty ≥ 3 on 1–5 scale). Derived from the paper's certainty scores — not an explicit paper metric.

↓ Lower is better
✅

Combined Utility

mean(Readability/10, SemanticMeaning/10, ROUGE-1 F1). Paper's exact formula. Measures how much narrative quality is preserved.

↑ Higher is better

⚠️ Important Distinction: Evidence Rate vs. Paper Metric

The paper reports certainty scores and uses certainty ≥ 3 as a filter to show 86.7% → 7.7%. We operationalize this as a continuous metric tracked across all architectures and rounds. This is our own analytical addition, not a metric the paper reports directly.

The 4 Pipelines We Designed

Each is a different answer to: how should multiple models collaborate to attack a profile?

Pipeline 0 — Baseline (Control)

Single GPT-4o Attacker

Replication of the original paper's setup. Single GPT-4o attacker → anonymize → re-attack. Needed as a reference point — without a baseline we cannot claim improvement.

Profile → GPT-4o Attack
→ Anonymizer (GPT-4o)
→ Re-Attack (GPT-4o)
→ Metrics
Pipeline 1 — Parallel

GPT-4o + Claude Simultaneously

Two LLMs attack the same profile independently at the same time. Different models have different blind spots — merging both perspectives gives the anonymizer a more complete threat picture.

Profile → [GPT-4o Attack] ─┐
        → [Claude Attack]  ─┤
                              → Merge
                              → Anonymizer
                              → Re-Attack Both
Pipeline 2 — Sequential

GPT-4o → Claude (Informed)

GPT-4o attacks first. Claude then receives GPT-4o's full reasoning and is told: "find what this missed." The second model is explicitly complementary, not redundant.

Profile → GPT-4o Attack
        ↓ [A's findings passed to B]
        → Claude Attack (informed)
        → Accumulated findings
        → Anonymizer → Re-Attack
Pipeline 3 — Evidence-Targeted

Priority-Guided Surgical Editing

Changes the anonymizer, not just the attack. Anonymizer receives priority labels (HIGH/MEDIUM/LOW) per attribute and must: (1) identify exact evidence phrases, then (2) replace only those phrases.

Parallel Attack → Priority Labels
  HIGH = both agree, certainty ≥ 3
  MED  = one attacker has evidence
  LOW  = both guessing
→ Evidence-Targeted Anonymizer
  Step 1: identify exact phrases
  Step 2: replace only those phrases
Pipeline 4 — Multi-Round Adversarial Loop

Iterative Attack–Anonymize Cycle (5 Rounds)

Directly implements the paper's core AA loop but with parallel multi-model attacks inside each round. Each round attacks the already-anonymized text, finding residual signals the previous round left behind. Ran for 5 rounds with stopping threshold at Top-3 accuracy ≤ 25%.

Round 1: Original Profile  → Parallel Attack → Anonymize → Anonymized_v1
Round 2: Anonymized_v1   → Parallel Attack → Anonymize → Anonymized_v2
Round 3: Anonymized_v2   → Parallel Attack → Anonymize → Anonymized_v3
Round 4: Anonymized_v3   → Parallel Attack → Anonymize → Anonymized_v4
Round 5: Anonymized_v4   → Parallel Attack → Anonymize → Anonymized_v5

✅ What Improved

  • Top-1 accuracy drops more aggressively in multi-model pipelines (−25pp sequential vs −10pp baseline)
  • Multi-round broke below the 55% Top-3 floor to 50%
  • Evidence rate reduced from 50% → 30% over 5 rounds (−20pp)
  • Utility perfectly stable across all 5 multi-round iterations at 0.937

⚠️ What Did Not Change

  • Top-3 adversarial accuracy: all single-round architectures converge to exactly 55%
  • Architecture complexity alone did not break the floor
  • Evidence-targeted: smallest Top-3 reduction (−5pp) despite most complex prompt

Why Defeat Rate and Adversarial Accuracy Tell Different Stories

Both baseline and parallel have 55% post-anonymization Top-3 accuracy — yet defeat rate is 15% vs 45%. This means: the baseline makes small improvements spread across all profiles (slightly fewer correct guesses everywhere), while the parallel makes complete improvements on some profiles (9 profiles fully protected) while leaving others similar. Same average, very different distribution of protection.

Core Finding

The 55% Floor

All four single-round architectures — regardless of how many attackers were used, how their perspectives were combined, or how targeted the anonymizer's edits were — converge to exactly 55% Top-3 adversarial accuracy post-anonymization. The multi-round loop is the only approach that breaks below this floor, reaching 50% while reducing the evidence rate from 50% to 30% over 5 rounds.

Why the Floor Exists

After phrase-level editing removes explicit PII signals (location names, job titles), the remaining 55% comes from demographic inference through writing style:

  • Vocabulary register — formal/informal, technical/lay — correlates with education and occupation
  • Topic selection — what subjects the person writes about — correlates with age, gender, marital status
  • Discourse structure — how arguments are made, what is presupposed — correlates with education
  • Relationship discourse — how partners, family, social dynamics are discussed — reveals marital status

These signals are distributed across every sentence. No attacker, however comprehensive, can force the anonymizer to rewrite the entire text without destroying utility. Only iteration erodes them gradually.

Alignment with the Original Paper

The paper shows 86.7% → 41.6% over 5 rounds, then → 7.7% when filtered to certainty ≥ 3. This means the paper's remaining 41.6% is mostly the same phenomenon — low-confidence stylistic inference that cannot be removed by targeted editing. Our results are weaker (55% floor vs 41.6%) due to smaller sample size (n=20 vs n=426) and synthetic rather than real profiles.

Why Our Numbers Differ from the Paper

  • Sample size: n=20 vs n=426. With n=20, one profile = 5% of the result — high statistical noise.
  • Synthetic vs real: Synthetic profiles may encode PII more explicitly, making them easier to anonymize at the phrase level.
  • Profile filter: We used certainty ≥ 1 (all profiles), the paper used harder profiles — explaining the lower 65% vs 86.7% starting accuracy.

Limitations

Honest assessment of what constrains our conclusions

📊

Small Sample Size (n=20)

Insufficient for statistical significance. Each profile represents 5% of results. One or two profiles changing could shift accuracy by 10pp. Results are directional, not definitive.

🤖

Synthetic Profiles

Synthetic data may encode PII more explicitly than real Reddit posts. Privacy claims on synthetic profiles have limited real-world validity.

⚖️

Utility Measurement

GPT-4o judged readability as 10/10 for all profiles — possibly too generous. LLM-based utility scoring is subjective and may not reflect real reader experience.

🔧

Single Anonymizer Model

Only the attacker architecture varies — anonymizer is always GPT-4o. A fairer study would also vary the anonymizer to isolate the effect of attack diversity.

Key Takeaways

What this thesis contributes

  • 1

    Parallel and sequential multi-model attacks do not improve Top-3 adversarial accuracy compared to a single model — all converge to 55%. However, they reduce Top-1 accuracy more aggressively (sequential: −25pp vs baseline: −10pp).

  • 2

    The evidence-targeted anonymizer successfully reduces Top-1 accuracy by 20pp (50%→30%) and specifically reduces Claude's evidence rate to 30%, confirming that evidence-grounded editing removes high-confidence signals. Probabilistic low-confidence guesses persist.

  • 3

    The multi-round loop is the only approach that breaks the 55% floor, reaching 50% from Round 1 and reducing evidence rate from 50% → 30% over 5 rounds. Utility stays perfectly stable at 0.937 throughout — iterative editing does not degrade narrative quality.

  • 4

    Model diversity matters for evidence elimination. Claude's evidence rate dropped to 30% post-anonymization vs GPT-4o's 50% in the parallel pipeline — the sociolinguistic perspective identifies different signal types that the anonymizer addresses differently.

  • 5

    This work directly answers an open question from the original paper. Section G.2 of Staab et al. explicitly identifies multi-model attacking as a future direction. Our experiments provide the first empirical answer to that question.
